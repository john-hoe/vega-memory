Getting Started

Install Vega Memory, point it at SQLite and Ollama, store a first memory, and wire the runtime into Cursor or Claude Code. This page uses the current repo scripts, config defaults, and Docker definitions.

Prerequisites

Node.js 18+, a writable local filesystem, and optional Ollama at http://localhost:11434.

Build Target

npm run build compiles TypeScript and copies the web assets.

API Mode

Set VEGA_API_KEY to enable authenticated HTTP access and the scheduler API.

1. Installation

Install dependencies, build the project, and expose the CLI locally.

git clone https://github.com/john-hoe/vega-memory.git
cd vega-memory
npm install
npm run build
npm link

2. Configuration

The repo ships .env.example for the core local settings. Add API-specific values when you want HTTP access or the scheduler.

Minimal Local Config

cp .env.example .env
VEGA_DB_PATH=./data/memory.db
VEGA_DB_ENCRYPTION=false
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=bge-m3

Enable HTTP API

VEGA_API_KEY=$(openssl rand -hex 16)
VEGA_API_PORT=3271
export VEGA_API_KEY VEGA_API_PORT
node dist/scheduler/index.js

3. Store Your First Memory

Verify the installation, then store and recall a concrete fact.

CLI

vega health

vega store "Always checkpoint WAL before backup" \
  --type pitfall \
  --project my-project

vega recall "sqlite backup" \
  --project my-project \
  --json

HTTP API

curl -X POST http://127.0.0.1:3271/api/store \
  -H "Authorization: Bearer $VEGA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Always checkpoint WAL before backup",
    "type": "pitfall",
    "project": "my-project"
  }'

4. MCP Setup

Use MCP for automatic tool calls in Cursor, or CLI rules for Claude Code.

Cursor

Add Vega as an MCP server in ~/.cursor/mcp.json.

{
  "mcpServers": {
    "vega": {
      "command": "node",
      "args": ["/absolute/path/to/vega-memory/dist/index.js"],
      "env": {
        "VEGA_DB_PATH": "/absolute/path/to/vega-memory/data/memory.db",
        "OLLAMA_BASE_URL": "http://localhost:11434",
        "OLLAMA_MODEL": "bge-m3"
      }
    }
  }
}

Then add a workspace rule that auto-calls session_start and memory_store as work happens.

Claude Code

Add the memory workflow to your project CLAUDE.md.

# Vega Memory Rules

## Session start
Run: `vega session-start --dir $(pwd) --json`

## Auto-store
- Task completed: `vega store "what was done" --type task_state --project PROJECT_NAME`
- Bug fixed: `vega store "error + solution" --type pitfall --project PROJECT_NAME`
- Decision made: `vega store "decision + reasoning" --type decision --project PROJECT_NAME`

## Before making changes
Run: `vega recall "relevant query" --project PROJECT_NAME --json`

## Session end
Run: `vega session-end --project PROJECT_NAME --summary "what was accomplished"`

5. Docker Deployment

The repository includes a multi-stage Dockerfile and a docker-compose.yml that starts Vega plus Ollama.

Compose

export VEGA_API_KEY=dev-api-key
docker compose up --build -d

curl -H "Authorization: Bearer $VEGA_API_KEY" \
  http://127.0.0.1:3000/api/health

The compose file binds Vega on port 3000 and mounts ./data into the container.

Runtime Notes

services:
  vega:
    command: ["node", "dist/scheduler/index.js"]
    ports:
      - "3000:3000"
  ollama:
    ports:
      - "11434:11434"

For production, replace the default API key, persist the volume, and front the service with your preferred network policy.
