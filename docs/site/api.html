HTTP API

This page documents the route surface currently registered in src/api/routes.ts. Examples are representative request and response payloads for the scaffolded API.

Authentication

When VEGA_API_KEY is configured, API routes require Authorization: Bearer <token>. Root API keys authorize globally, tenant API keys scope requests to a single tenant, and /api/billing/webhook is intentionally exempt from auth.

Header

{
  "Authorization": "Bearer YOUR_VEGA_API_KEY",
  "Content-Type": "application/json"
}

Error Response

{
  "error": "unauthorized"
}

Memory CRUD

Store, list, update, and delete memory records.

POST /api/store

Create a new memory entry with content, type, and optional metadata.

Request JSON
{
  "content": "Use WAL mode for concurrent SQLite access.",
  "type": "decision",
  "project": "vega-memory",
  "title": "SQLite concurrency default",
  "tags": ["sqlite", "decision"],
  "importance": 0.92,
  "source": "auto"
}
Response JSON
{
  "id": "mem_123",
  "type": "decision",
  "project": "vega-memory",
  "title": "SQLite concurrency default",
  "content": "Use WAL mode for concurrent SQLite access.",
  "importance": 0.92,
  "source": "auto",
  "tags": ["sqlite", "decision"],
  "created_at": "2026-04-08T00:00:00.000Z",
  "updated_at": "2026-04-08T00:00:00.000Z"
}
GET /api/list

List stored memories, optionally filtered by project, type, tenant, limit, and sort order.

Request JSON
{
  "query": {
    "project": "vega-memory",
    "type": "decision",
    "limit": 20,
    "sort": "updated_at"
  }
}
Response JSON
[
  {
    "id": "mem_123",
    "type": "decision",
    "project": "vega-memory",
    "title": "SQLite concurrency default",
    "content": "Use WAL mode for concurrent SQLite access.",
    "tags": ["sqlite", "decision"]
  }
]
PATCH /api/memory/:id

Update mutable memory fields including content, importance, and tags.

Request JSON
{
  "path": { "id": "mem_123" },
  "body": {
    "content": "Use WAL mode plus checkpointing before backup.",
    "importance": 0.95,
    "tags": ["sqlite", "backup", "decision"]
  }
}
Response JSON
{
  "id": "mem_123",
  "action": "updated"
}
DELETE /api/memory/:id

Delete a memory record by identifier.

Request JSON
{
  "path": { "id": "mem_123" }
}
Response JSON
{
  "id": "mem_123",
  "action": "deleted"
}

Search

Recall memories by similarity, stream results, and inspect service health.

POST /api/recall

Run a ranked recall query against the memory corpus.

Request JSON
{
  "query": "sqlite backup",
  "project": "vega-memory",
  "type": "pitfall",
  "limit": 5,
  "min_similarity": 0.3
}
Response JSON
[
  {
    "id": "mem_987",
    "type": "pitfall",
    "project": "vega-memory",
    "content": "Checkpoint WAL before copying SQLite files.",
    "similarity": 0.91,
    "finalScore": 0.94
  }
]
GET /api/recall/stream

Stream recall results over server-sent events using the same query shape as /api/recall.

Request JSON
{
  "query": {
    "query": "tenant isolation",
    "project": "vega-memory",
    "limit": 3,
    "min_similarity": 0.25
  }
}
Event Data
{
  "id": "mem_321",
  "type": "decision",
  "content": "Block tenant bearer keys from admin routes.",
  "similarity": 0.88,
  "finalScore": 0.9
}

Sessions

Load relevant context at the start of work and summarize outcomes at the end.

POST /api/session/start

Collect relevant memories, warnings, and wiki references for a working directory.

Request JSON
{
  "working_directory": "/Users/johnmacmini/workspace/vega-memory",
  "task_hint": "T92 docs scaffold"
}
Response JSON
{
  "project": "vega-memory",
  "active_tasks": [],
  "preferences": [],
  "context": [],
  "relevant": [],
  "relevant_wiki_pages": [],
  "wiki_drafts_pending": 0,
  "recent_unverified": [],
  "conflicts": [],
  "proactive_warnings": [],
  "token_estimate": 944.75
}
POST /api/session/end

Record a session summary and optional completed task identifiers.

Request JSON
{
  "project": "vega-memory",
  "summary": "Built docs/site scaffold and verified build + tests.",

}
Response JSON

  "project": "vega-memory",
  "action": "ended"

Admin

Administrative routes require an admin-capable context. This section also covers billing and webhook operations exposed by the same router.

GET/api/analytics

Retrieve aggregated usage statistics, optionally filtered by since.

Request JSON
{
  "query": { "since": "2026-04-01T00:00:00.000Z" }
}
Response JSON
{
  "total_memories": 1024,
  "active_projects": 18,
  "recent_sessions": 44
}
POST/api/compact

Run memory compaction for the full workspace or a single project.

Request JSON
{
  "project": "vega-memory"
}
Response JSON
{
  "scanned": 120,
  "merged": 8,
  "archived": 2
}
GET/api/admin/audit

List audit entries with actor, action, time, tenant, and pagination filters.

Request JSON
